# GitHub 网页上传文件详细步骤

由于你的电脑没有安装 Git，我们将使用 GitHub 网页界面上传文件。请按照以下步骤操作：

## 第一步：访问你的仓库

1. 打开浏览器，访问：https://github.com/kj0204/nunzhaku2.8
2. 登录你的 GitHub 账号

## 第二步：上传 .github 文件夹

### 方法 A：使用 GitHub Desktop（推荐，如果已安装）

如果你安装了 GitHub Desktop：
1. 打开 GitHub Desktop
2. 点击 "File" -> "Add Local Repository"
2. 选择 `D:\NEW电脑共享\新建文件夹` 文件夹
3. 点击 "Publish repository" 按钮发布到 GitHub

### 方法 B：使用网页界面上传（如果未安装 GitHub Desktop）

由于 GitHub 网页界面不支持直接上传文件夹，我们需要分别上传文件：

#### 上传 workflow 文件：

1. 在仓库页面，点击 "Add file" -> "Create new file"
2. 在文件名输入框中输入：`.github/workflows/build_nunchaku.yml`
3. 打开文件：`D:\NEW电脑共享\新建文件夹\.github\workflows\build_nunchaku.yml`
4. 复制文件内容，粘贴到 GitHub 的编辑框中
5. 滚动到页面底部，在 "Commit new file" 部分填写：
   - Commit message: `Add GitHub Actions workflow for building Nunchaku with PyTorch 2.8`
6. 点击绿色按钮 "Commit new file"

#### 上传 README 文件：

1. 点击 "Add file" -> "Create new file"
2. 在文件名输入框中输入：`README.md`
3. 打开文件：`D:\NEW电脑共享\新建文件夹\README.md`
4. 复制文件内容，粘贴到 GitHub 的编辑框中
5. 滚动到页面底部，在 "Commit new file" 部分填写：
   - Commit message: `Add README documentation`
6. 点击绿色按钮 "Commit new file"

#### 上传修改脚本（可选）：

1. 点击 "Add file" -> "Create new file"
2. 在文件名输入框中输入：`modify_torch_version.py`
3. 打开文件：`D:\NEW电脑共享\新建文件夹\modify_torch_version.py`
4. 复制文件内容，粘贴到 GitHub 的编辑框中
5. 滚动到页面底部，在 "Commit new file" 部分填写：
   - Commit message: `Add torch version modification script`
6. 点击绿色按钮 "Commit new file"

## 第三步：启用 GitHub Actions

1. 在仓库页面，点击顶部的 "Actions" 标签
2. 如果看到提示 "I understand my workflows, go ahead and enable them"，点击它

## 第四步：触发编译

1. 在 "Actions" 页面，左侧选择 "Build Nunchaku with PyTorch 2.8"
2. 点击右侧的 "Run workflow" 按钮
3. 在弹出的窗口中，确认参数（或修改）：
   - PyTorch version: `2.8.0`
   - Python version: `3.12`
   - CUDA version: `12.8`
4. 点击绿色按钮 "Run workflow"

## 第五步：等待编译完成

1. 编译过程大约需要 15-30 分钟
2. 你可以点击工作流运行记录查看进度
3. 等待状态变为绿色对勾（表示成功）

## 第六步：下载编译好的 wheel 文件

1. 在 "Actions" 页面，点击最新的工作流运行记录
2. 滚动到页面底部
3. 在 "Artifacts" 部分，找到 `nunchaku-wheel-py3.12-torch2.8.0`
4. 点击下载按钮
5. 解压下载的 zip 文件
6. 里面就是编译好的 `.whl` 文件

## 第七步：安装 wheel 文件

```bash
pip install nunchaku-1.1.0+torch2.8-cp312-cp312-win_amd64.whl
```

## 常见问题

### Q: 上传文件时提示文件名无效？
A: 确保 `.github/workflows/build_nunchaku.yml` 路径正确，特别是前面的点号 `.`

### Q: 编译失败怎么办？
A: 查看工作流运行记录中的日志，通常会显示具体的错误信息

### Q: 下载的 artifacts 保留多久？
A: GitHub 会保留 90 天，之后会自动删除
